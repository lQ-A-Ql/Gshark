GShark-Sentinel 前端 UI/UX 设计文档
1. 设计概述与技术选型
产品定位：面向网络安全与 CTF 竞赛的离线 PCAP 流量审计工作站。

前端栈：Vue3 (或 React) + TypeScript + TailwindCSS。

设计原则：

高信息密度：摒弃大片留白，最大化展示数据包细节。

上下文联动：列表、解析树、字节码视图必须做到毫秒级同步联动。

无感渲染：利用虚拟滚动（Virtual List）支撑百万级 DOM 渲染。

2.2 HTTP 专用流追踪器 (HTTP Stream Tracker) - [新增]
功能定位：自动剥离底层 TCP 握手与确认包，专门针对包含 HTTP 协议的流进行 Request/Response 重组。屏蔽非 HTTP 的纯 TCP/UDP 流接入。

UX 说明：

结构化分离：强制将 Request 和 Response 上下或左右分屏展示。

智能解码：针对 Response Body，自动识别 Content-Encoding: gzip 并解压，自动依据 Content-Type 提供 JSON 格式化、HTML 渲染或图片预览。

拦截机制：如果用户对 SSH、FTP 等纯 TCP 流点击“HTTP 追踪”，系统应提示“当前 TCP 流未检测到有效 HTTP 载荷”，并引导至通用 TCP 流视图。
