
## 1. 项目概述

### 1.1 产品定位

GShark-Sentinel 是一款专为网络安全从业人员、CTF 选手及应急响应专家打造的桌面端 PCAP/PCAPNG 离线流量分析工具。它通过封装 `tshark` 提供底层解析能力，旨在解决传统 Wireshark 在大规模流量筛查效率低下、CTF 专项提取繁琐、解密配置不友好等痛点。

### 1.2 技术选型

- **前端展示**：Vue3 (或 React) + TypeScript + TailwindCSS + Element Plus/Radix UI。
- **桌面端桥接**：Wails (提供原生窗口管理与 Go-JS IPC 通信)。
- **后端引擎**：Go 1.21+。
- **解析核心**：本地环境的 `tshark` (Wireshark 命令行工具)。

## 2. 核心功能需求

### 2.1 基础视窗与底层解析

- **大文件加载与流式解析**：支持加载 GB 级别的离线数据包。Go 后端需调用 `tshark -T ek` 或 `-T json` 边解析边通过 Wails Events 推送给前端。
- **百万级虚拟滚动视图**：前端数据包列表必须采用虚拟滚动（Virtual List）技术，确保在数十万条记录下依然保持 60fps 的丝滑滚动与无延迟渲染。
- **视图三向联动**：数据包列表、协议解析树、十六进制/ASCII 视图需实现精确联动。点击协议树某字段，十六进制视图需**自动高亮**对应字节；反向亦然。
- **全局条件过滤**：顶部提供带语法高亮和历史记录的过滤输入框，全面兼容标准 `tshark` / Wireshark 过滤语法。

### 2.2 OWASP Top 10 专项审计 (Security Hunting)

在顶部菜单栏提供一键式威胁筛查入口，触发后自动填充过滤条件或调出分析面板：

- **注入与 RCE 检测**：内置高频正则字典，一键筛选 SQL 注入（`union`, `select`）、XSS（`<script>`）与命令执行（`whoami`, `/etc/passwd`）特征。
- **异常扫描分析**：统计并高亮显示短时间内产生大量 404/403 状态码的源 IP，识别目录爆破与漏洞扫描行为。
- **敏感信息泄露**：通过正则匹配自动在 HTTP 响应报文中提取手机号、身份证号、AK/SK、JWT Token 等敏感数据。

### 2.3 CTF 竞赛专项提效 (CTF Toolkit)

- **Flag 全局嗅探**：支持自定义前缀（如 `flag{`, `ctf{`），并发搜索全文及其 Base64、Hex 编码变体，命中结果在独立面板聚合展示。
- **智能附件提取 (Object Export)**：一键扒取 HTTP/FTP 流中的文件。前端以“抽屉视图 (Drawer)”呈现，支持图片直接渲染缩略图，并提供按文件类型、大小过滤的功能。
- **非标准协议画像**：自动识别端口隐蔽流量（如 8888 端口的 HTTP，53 端口的非 DNS 隧道流量）。
- **隐写术初筛**：自动检查提取出的 PNG/JPG 文件是否存在 IHDR 错误或文件尾 (EOF) 后附带隐藏数据。

### 2.4 现代流追踪体系 (Stream Tracker)

- **HTTP 专用会话重组**：针对 HTTP 流提供专用的双屏（Request/Response）分析界面。自动解压 Gzip 响应体，自动对 JSON/HTML 进行格式化，支持一键 URL/Base64 解码。
- **Raw TCP/UDP 流追踪**：针对非 HTTP 流量（如反弹 Shell、SSH、私有二进制协议），提供红蓝双色的原始数据流展示，支持 ASCII 与 Hex 视图快速切换。

### 2.5 TLS / HTTPS 流量一键解密

- **密钥日志加载**：支持一键导入浏览器生成的 `SSLKEYLOGFILE`，自动重启解析隧道还原 TLS 1.3 现代加密流量。
- **私钥加载**：支持导入 RSA 私钥（PEM），绑定目标 IP 与端口，针对 TLS 1.2 及以下不具备前向安全性的流量进行解密。

### 2.6 开放式插件引擎

- **策略模式挂载**：允许用户通过 Go DLL (或集成 Lua VM) 编写自定义解析逻辑。
- **插件能力开放**：插件需能读取结构化 Packet 数据，支持在 UI 上打标签（如“发现冰蝎流量”）、解密特定 Payload，或输出调试日志。

## 3. 非功能性需求与 UX 规范

### 3.1 性能指标

- **首屏响应**：加载 100MB 以内的 PCAP，首批数据列表展示时间不超过 2 秒。
- **内存控制**：前端内存占用不应随数据量线性暴增，需依靠后端的游标分页或按需推送机制限制 JS 堆内存。

### 3.2 交互设计规范

- **高信息密度**：遵循 IDE 设计范式，减少留白。界面支持多面板拖拽分隔（Splitter）。
- **暗黑模式支持**：原生支持 Dark/Light 主题切换，贴合安全分析师使用习惯。
- **键盘快捷键**：为高频操作（如打开文件 `Ctrl+O`、追踪流 `Ctrl+Shift+T`、聚焦过滤栏 `Ctrl+F`）绑定全局快捷键。

| ---------------- | ----------------------- | ------------------------------------------------------------ |
|  |    |       |
| **桥接层 (IPC)** | Wails Bindings & Events | 负责 Go 与 JS 之间的高效数据通信，分发事件流。               |

| **底层核心**     | `tshark` 可执行文件     | 执行深度协议解包，应用 BPF 和 Display Filter 过滤。          |
